# VideoBrick Pipeline (без Figma и без платных MCP лимитов)

## Цель

Быстро и стабильно выпускать визуально сильные версии лендинга `VideoBrick` без зависимости от Figma MCP.

## Принцип

Code-first: дизайн формируется сразу в коде (`index.html` + CSS/JS), а нейросети используются для контента и визуалов.

## Структура артефактов

- `index.html` — рабочий интерфейс и логика.
- `assets/img/` — финальные изображения для сайта.
- `assets/raw/` — исходники из генераторов (Nano Banana и т.д.).
- `prompts/` — текстовые промпты для генерации изображений и текстов.
- `docs/DECISIONS.md` — короткие решения по стилю, сетке, тону и ограничениям.

## Этапы работы

1. Контент-каркас:
- Утвердить структуру страницы (hero, proof, process, CTA).
- Утвердить копирайт в черновом виде.

2. Генерация визуалов:
- Сгенерировать изображения через Nano Banana по заранее сохраненным промптам.
- Отобрать 1-2 лучших варианта на блок.
- Сохранить в `assets/raw/`, затем финальные версии в `assets/img/`.

3. Сборка UI в коде:
- Встроить финальные визуалы в `index.html`.
- Привести типографику, отступы, цвета и кнопки к единому стилю.
- Сразу проверять мобильную версию.

4. Качество:
- Проверка адаптива: 390px, 768px, 1280px.
- Проверка читабельности текста на всех фоновых изображениях.
- Проверка базовой доступности (контраст, фокус, подписи форм).

5. Фиксация контекста:
- После каждой итерации обновлять `SESSION_SUMMARY_YYYY-MM-DD.md`.
- Кратко фиксировать: что изменили, что откатили, что делаем следующим шагом.

## Правила, чтобы не развалить процесс

- Не менять одновременно логику формы и визуальный стиль в одном проходе.
- Не генерировать новые визуалы, пока не определена структура блоков.
- Все финальные ассеты хранить в репо, не ссылаться на временные внешние URL.
- Каждый новый чат начинать с чтения `SESSION_SUMMARY` и этого файла.

## MVP-ритм итераций

- Итерация 1: структура + базовый визуальный стиль.
- Итерация 2: сильный hero + согласованные ассеты.
- Итерация 3: UI-полировка + mobile-first исправления.
- Итерация 4: финальные тексты и подготовка к публикации.
