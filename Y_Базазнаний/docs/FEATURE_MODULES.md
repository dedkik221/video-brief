# Границы модулей (чтобы не ломать рабочие фичи)

## Принцип
Каждая фича — отдельный модуль с собственной разметкой, стилями и логикой.
Изменения внутри модуля не должны требовать правок в других модулях.

## Модуль A: Knowledge Base
- Папка: `site/knowledge/`
- Ответственность: статьи, шаблоны процессов, обучающие материалы.
- Нельзя: тянуть в этот модуль логику создания заявок.

## Модуль B: Request Wizard
- Папка: `site/request/`, `site/js/request-wizard.js`
- Ответственность: сбор параметров заказа, валидация, локальный preview.
- Нельзя: зависеть от структуры статей базы знаний.

## Модуль C: Shared UI
- Папка: `site/styles.css`
- Ответственность: общие переменные, типографика, базовые компоненты.
- Нельзя: feature-специфичные хаки в глобальных стилях.

## Подготовка к интеграции
- Слой интеграции добавляется позже как отдельный модуль (например `site/js/integration/`).
- До интеграции мастер заявки работает с локальным JSON-результатом.
